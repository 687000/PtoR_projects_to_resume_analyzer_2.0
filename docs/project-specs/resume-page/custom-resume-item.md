# Custom Resume Item Feature Design

**Feature:** Tailored Resume Content View and Editor
**Page:** Resume Page — Detail View

---

## 1. Overview

The Custom Resume Item is the detail view for a single JD target. It shows the full tailored resume output generated by matching a job description against the user's saved projects. The user can edit bullets inline, reorder projects, pin specific bullets for export, and copy or export the final resume section.

This view is opened from the Resume List by clicking "View Resume" on any saved JD target card.

**Key Goals:**
- Show all matched projects ranked by fit score in one consolidated view
- Allow fine-grained editing: pick and reorder bullets, adjust framing
- Produce a ready-to-paste resume block without leaving the app

---

## 2. User Flow

### View and Edit Tailored Content

1. User opens a JD target from the Resume List
2. Detail view opens showing:
   - JD summary panel at top (role title, company, extracted requirements)
   - Ranked project sections below, one per matched project
3. Each project section shows:
   - Project title and fit score
   - Which JD requirements this project addresses (requirement tags)
   - 2–4 tailored resume bullets, editable inline
4. User edits bullets as needed:
   - Click a bullet to edit text inline
   - Drag to reorder bullets within a project
   - Toggle a bullet on/off (checked bullets are included in export)
5. User can drag entire project sections to reorder projects in the final resume
6. User clicks "Copy Resume Section" → clipboard receives the formatted bullet block

### Re-run Matching

1. User clicks "Re-match" at top of detail view
2. Matching runs again against current project pool
3. Results update in place (with confirmation if existing edits would be lost)

### Export

1. User clicks "Export" → modal with format options (plain text, Markdown, clipboard)
2. Only checked/pinned bullets are included
3. Output groups bullets under project titles or flattens to a single list (user choice)

## 3. UI Design

### Layout

```
[← Back to Job Targets]

[JD Summary Panel]
Role: Senior Frontend Engineer · Acme Corp
Requirements extracted:
  [Vue 3] [TypeScript] [cross-team] [web architecture] [3+ yrs]
[Re-match]  [Edit JD Requirements]

────────────────────────────────────────────────────

[Project Section #1]  Fit Score: 92%
Project A — Workflow Platform Rebuild
Addresses: [Vue 3] [web architecture] [cross-team]

  ☑ • Led frontend architecture for a multi-team workflow platform...
  ☑ • Defined component API contracts across web and mobile surfaces...
  ☐ • Coordinated with PM to scope feature delivery across 3 sprints...
  [+ Add bullet]

[Project Section #2]  Fit Score: 74%
Project B — Internal Admin Tool
Addresses: [Vue 3] [TypeScript]

  ☑ • Built modular form system using Vue 3 Composition API...
  ☑ • Reduced page load time by 40% via lazy-loaded route chunks...
  [+ Add bullet]

────────────────────────────────────────────────────

[Copy Resume Section]  [Export ▾]  [Save Changes]
```

### Interaction Details

- **Bullet toggle (☑/☐):** Controls inclusion in export; unchecked bullets are greyed but not deleted
- **Inline edit:** Click any bullet text to activate a textarea; blur to save
- **Drag handle:** Left of each bullet row for reordering within a project section
- **Project section drag:** Header row has a drag handle for reordering project sections
- **"+ Add bullet":** Opens an empty editable row; user can type a custom bullet or pick from the full `resume_bullets[]` list for that project via a dropdown
- **"Generate Bullets" button:** Per project section — triggers Stage 3 (LLM generation using `jd-tailored-bullets-prompt`). Generates 2-3 new tailored bullets that address JD requirements while tracing back to project evidence. Only available when `ANTHROPIC_API_KEY` is set. Generated bullets are appended to the list with `source: 'generated'` and include detailed metadata (addresses, source_references, reasoning).
- **Requirement tags:** Each tag is clickable — clicking highlights which bullets address that requirement
- **Seniority fit badge:** Displayed in the JD summary panel — shows `meets` / `partial` / `below` with the `seniority_fit_reason` as a tooltip

### JD Summary Panel

Displays extracted requirements as chips/tags. User can:
- Remove a tag (to exclude it from matching influence)
- Add a custom tag manually
- Click "Edit JD Requirements" to open a full text editor for the extracted requirement list

## 4. Technical Implementation

### Data Model: JD Target Entity

```
JDTarget
├── id                        (uuid)
├── created_at                (ISO 8601 UTC)
├── updated_at                (ISO 8601 UTC)
├── role_title                (extracted from JD)
├── company                   (extracted or null)
├── raw_jd_text               (full extracted source text)
├── source_metadata           (filename | url | pages | size)
├── seniority_fit             ('meets' | 'partial' | 'below')
├── seniority_fit_reason      (string — one sentence explanation)
├── extracted_requirements
│   ├── tech_stack[]          (string[])
│   ├── collaboration[]       (string[])
│   ├── domain[]              (string[])
│   └── seniority_requirement
│       ├── min_years         (int | null)
│       ├── expected_level    ('junior' | 'junior-mid' | 'mid' | 'senior' | 'lead' | 'principal' | 'not_stated')
│       └── ownership_signals[] (string[])
├── matched_projects[]
│   ├── project_id            (ref to Project entity)
│   ├── fit_score             (0–100)
│   ├── fit_reason            (string — one sentence from LLM, or empty in fallback)
│   ├── addressed_requirements[] (subset of tech_stack + domain + collaboration)
│   └── tailored_bullets[]
│       ├── bullet            (string — the resume bullet text)
│       ├── included          (bool — pinned for export)
│       ├── source            ('selected' | 'generated')
│       ├── addresses         (array — ['tech_stack', 'domain', 'collaboration', 'seniority'])
│       ├── source_references (array — evidence links to project content)
│       ├── reasoning         (string — why this bullet addresses JD requirements)
│       └── order             (int — display order)
└── last_exported_at          (ISO 8601 UTC | null)
```

**Notes:**
- `seniority_fit` and `seniority_fit_reason` are computed once per JD by comparing the JD's `seniority_requirement` against `data/user_profile.json`. They are not re-evaluated per project.
- `tailored_bullets` contains bullets **selected from existing project content** during initial matching (Stage 2). New bullets generated on demand via "Generate Bullets" (Stage 3) are appended with `source: 'generated'` and include detailed metadata.
- `fit_score` reflects tech + domain + collaboration match only — seniority is not part of per-project scoring.
- Generated bullets use the `jd-tailored-bullets-prompt` and require `ANTHROPIC_API_KEY` to be configured.

### Key Components

**JDSummaryPanel.vue:**
- Displays extracted requirements as editable tag chips
- "Re-match" and "Edit JD Requirements" actions

**MatchedProjectSection.vue:**
- One instance per matched project
- Fit score badge, requirement tag highlights
- Bullet list with toggle, inline edit, drag reorder
- "Add bullet" with optional picker from project's full bullet pool

**ResumeExporter.vue:**
- Collects all `included: true` bullets across all project sections in display order
- Formats as plain text, Markdown, or clipboard payload
- Handles flat list vs. grouped-by-project toggle

### Performance Targets

- Detail view load: < 1 second (data already computed at match time)
- Inline bullet save: optimistic update, persist to store async
- Export generation: < 500ms

## 5. Validation and Testing

### User Acceptance Criteria

- [ ] Detail view loads with correct JD summary and ranked matched projects
- [ ] Bullets can be toggled, edited inline, and reordered by drag
- [ ] Project sections can be reordered by drag
- [ ] "Add bullet" allows typing a custom bullet or picking from the project's full list
- [ ] "Copy Resume Section" copies only checked bullets in display order
- [ ] Edits persist on navigation away and return
- [ ] Re-match updates results; warns if unsaved edits exist

### Edge Cases

- **Zero matched projects:** Show "No projects matched this JD. Add more projects to improve results." with a link to the Projects Page
- **Project deleted after match:** Show "[Project deleted — bullets retained for reference]" section in a muted style; allow the user to keep or discard those bullets
- **All bullets unchecked:** "Copy / Export" button disabled with tooltip "Check at least one bullet to export"
- **User-edited bullet on re-match:** Re-match does not overwrite bullets marked `source: 'user'`; it only regenerates `source: 'generated'` bullets

## 6. Future Enhancements

- Role-aligned bullet rewriting: toggle between Vue developer framing, middle office framing, etc. per bullet
- Bullet history: undo edits back to the generated original
- Side-by-side comparison: view the raw JD requirement next to the bullet addressing it
- Resume builder: arrange multiple JD target outputs into a full resume document
